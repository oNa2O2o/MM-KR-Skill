## Phase 0 · 初始化

### Step 0：自动更新检查

每次调用 `/MM-KR` 时，**必须先执行更新检查**，再做任何其他操作：

1. 读取本地版本文件 `MM-KR-参考/version.json`，获取 `version` 字段
2. 用 `curl` 静默获取远程版本：
   ```bash
   curl -sf --connect-timeout 5 "https://raw.githubusercontent.com/oNa2O2o/MM-KR-Skill/main/version.json"
   ```
3. 比较本地和远程的 `version` 字段（语义化版本比较）
4. **三种结果**：
   - 远程获取失败或超时 → 静默跳过，输出 `[更新检查] 无法连接远程仓库，使用本地版本 vX.X.X`，继续流程
   - 版本相同 → 静默跳过，输出 `[更新检查] 当前已是最新版本 vX.X.X`，继续流程
   - 远程版本更高 → **暂停流程**，向用户展示：
     ```
     ⚡ MM-KR 有新版本可用
     当前版本：vX.X.X
     最新版本：vY.Y.Y
     更新内容：[远程changelog字段]
     
     是否立即更新？（更新/跳过）
     ```
5. 用户选择"更新" → 执行更新流程（见下方）
6. 用户选择"跳过" → 继续使用当前版本

### 更新执行流程

用户确认更新后，执行以下步骤：

```bash
# 1. 备份当前版本到临时目录
BACKUP_DIR=$(mktemp -d)
SKILL_DIR="<.claude/commands所在绝对路径>"
cp -r "$SKILL_DIR/MM-KR.md" "$SKILL_DIR/MM-KR-参考" "$BACKUP_DIR/"

# 2. 从GitHub下载最新文件
REPO_RAW="https://raw.githubusercontent.com/oNa2O2o/MM-KR-Skill/main"
FILES=(
  "MM-KR.md"
  "MM-KR-参考/version.json"
  "MM-KR-参考/画风约束.md"
  "MM-KR-参考/情绪导演模板.md"
  "MM-KR-参考/huggingfield模板.md"
  "MM-KR-参考/api2img-setup.md"
  "MM-KR-参考/deepwhite-编剧引擎.md"
  "MM-KR-参考/deepwhite-图片提示词模板.md"
  "MM-KR-参考/deepwhite-分镜提示词模板.md"
)
FAIL=0
for f in "${FILES[@]}"; do
  mkdir -p "$SKILL_DIR/$(dirname "$f")"
  curl -sf "$REPO_RAW/$f" -o "$SKILL_DIR/$f" || FAIL=1
done

# 3. 验证更新结果
if [ $FAIL -eq 1 ]; then
  echo "部分文件下载失败，回滚到备份版本"
  cp -r "$BACKUP_DIR/MM-KR.md" "$BACKUP_DIR/MM-KR-参考" "$SKILL_DIR/"
else
  echo "✅ 更新成功"
  # 读取新版本号确认
  cat "$SKILL_DIR/MM-KR-参考/version.json"
fi
rm -rf "$BACKUP_DIR"
```

更新完成后输出：`[更新完成] MM-KR 已更新到 vY.Y.Y`，然后继续正常流程。

## Phase 1 · 剧情·脚本·标题 ⛔ 用户说"合格"才继续

并列输出两个版本，用户选择或合并使用。

### 版本A · 快速简案（默认）

每个角色产出一个完整文档 `XX线_完整文档.md`，包含：

#### 1.1 剧情大纲
- 身份、性格、好感度走向（数值三段递进）
- 3场景概要（每场景一句话）

#### 1.2 视频脚本（×3场景）
- 情节描述
- 韩语台词原文（直接可表演，禁止内心戏）
- 中文翻译
- 结尾动作

#### 1.3 标题与标签
- 视频大标题备选×3（韩语+中文）
- 每幕小标题备选×3（韩语+中文）
- 标签（韩语）

#### 1.4 后期UI文案（×3场景）
- 场景标签：韩语 / 中文（X일차 - 地点 / 第X天 - 地点）
- 好感度：호감도 X / 好感度 X
- 问题文字：韩语 / 中文
- 选项A/B：韩语 / 中文

### 版本B · DeepWhite编剧版

基于 deepwhite-screenwriting-v1（山音+screenwriter引擎），额外产出：

- **控制性理念**：`生活变成X，因为主角做了Y，而原因是Z`
- **人物深度**：Want / Need / Arc，人物矛盾性
- **场景价值转变**：每场标注进入价值→退出价值，价值对（控制/失控、信任/背叛等）
- **因果性审计**：逐场检查`第N场发生，是因为第N-1场发生了什么`
- **双轨节奏标注**：情节松紧 × 情感轻重
- **量化评分**：开场钩子/视觉概念/场景因果/人物主动性/台词克制/结尾钩子/节奏，各10分
- **自检清单**：山音自检 + screenwriter审计

格式见 `MM-KR-参考/deepwhite-编剧引擎.md`

### 通用规则
- 单角色单线，3段式情绪递进，不混线
- 台词必须直接可表演，不写内心戏、不写心理描写
- 只写能被摄影机拍到、麦克风听到的内容
- 每个视频一个文档，不分文档

## Phase 2 · 人设图 ⛔ 用户确认

1. 使用 **image edit API**（`/v1/images/edits`），用户原图作输入
2. 提示词模板：`保持这个角色的外貌特征和画风不变。生成这个角色的人设参考图，白色背景，包括：左侧全身站姿，右上面部五官大特写，【剧情对应的】三个表情，右下角一排配色色块。不要任何文字。`
3. 分辨率：4K（`1536x1536` 或 API支持的最大正方形尺寸）
4. 输出到 `人设图/角色名_人设图.png`

### 附加：DeepWhite双语图片提示词

生成人设图的同时，额外输出一份双语提示词文档 `人设图/角色名_图片提示词.md`：
- 结构：`[主体+动作] + [场景] + [构图] + [光影] + [风格] + [镜头] + [色调]`
- English Prompt + 中文提示词，嵌入韩式厚涂固定参数
- 供用户在其他平台/模型手动生成使用
- 格式见 `MM-KR-参考/deepwhite-图片提示词模板.md`

---

## Phase 3 · 分镜图 ⛔ 用户确认

1. 使用 **image edit API**，人设图作输入
2. 根据场景的动作变化拆分关键帧数量（通常3-6帧）
3. 所有帧按9:16比例排列到一张图上，4K分辨率
4. 提示词要求：
   - 保持角色外貌和画风不变
   - 描述每个帧的场景/动作/表情
   - **服装描述必须逐项对照人设图**，不凭记忆
   - 禁止夸张瞪眼等AI味表情
5. 输出到 `视频N_角色名/分镜图/分镜_场景X.png`

**画风固定参数**：韩式厚涂、暗朦调、泛朦、低饱和、反射、质感、泛光模糊晕染
详见 `MM-KR-参考/画风约束.md`

### 附加：DeepWhite双语图片提示词

每张分镜图同步输出双语提示词，写入对应 Seedance参考 文件夹：
- 每帧一组 English Prompt + 中文提示词
- 嵌入韩式厚涂固定参数 + 场景光影 + 构图 + 色调
- 格式见 `MM-KR-参考/deepwhite-图片提示词模板.md`

## Phase 4 · UI素材 ⛔ 用户确认

### Step 1：UI参考图
- 用分镜图作为背景，叠加游戏风格UI
- UI元素：场景标签（左上）、好感度（右上）、问题文字（底部）、选项按钮×2（底部）
- 输出到 `UI/UI参考.png`
- 等用户确认

### Step 2：UI绿幕提取
- 使用 **image edit API**，UI参考图作输入
- **固定提示词**：`保持图片中的UI元素配色大小比例完全不变，将每个UI元素单独提取放置到绿幕背景上`
- 输出到 `UI/UI绿幕模板.png`
- UI全局复用，不带文案

## Phase 5 · Seedance提示词 ⛔ 用户确认

### 产出
每个视频一个文档，放在 `视频N_角色名/Seedance参考/` 文件夹内，与参考图同目录。

### 时长计算
韩语台词音节数 ÷ 6~8 = 朗读秒数，加上动作/停顿缓冲 → 目标8-12秒，单条上限15秒。在提示词中标注建议时长。

### 并列三版

**版本A · 情绪导演六段式**
- 格式见 `MM-KR-参考/情绪导演模板.md`
- 六段：视听限制 / 语言台词 / 运镜手法 / 风格色调与光景 / 角色与场景设定 / 时间轴详细叙事
- 总字数 ≤1900字
- 中文

**版本B · HuggingField JSON**
- 格式见 `MM-KR-参考/huggingfield模板.md`
- JSON数组：`[{"lang":"en","prompt":"..."},{"lang":"zh","prompt":"..."}]`
- ZH ≤1800字符
- 分为：Style & Mood / Dynamic Description / Static Description / Audio

**版本C · DeepWhite分镜版**
- 格式见 `MM-KR-参考/deepwhite-分镜提示词模板.md`
- 固定开头 → 风格四段（全局画质/人物材质/灯光与风格/核心特效）→ 素材句柄 → 空间警告 → 镜头段落 → 环境音效 → 防错警告
- 每镜头字段：画面动作概述 → 画面构图 → 机位 → 动作 → 音效
- 摄影机与情绪同步（焦点人物情绪→摄影机类型映射）
- 表演微节拍：禁止泛情绪标签，拆为肌肉/呼吸/眼神/皮肤/姿态
- 台词前后微表演（吞咽→台词→重音→停顿）
- 每条 ≤2200中文字符
- 中文

### 通用规则（三版共用）
- 镜头自然晃动模拟手持拍摄
- [图1]人设图 [图2]分镜图 引用格式（版本C中对应@image1/@image2）
- 禁止引用拟声词音节
- 无比喻修辞，纯物理描述
- 服装描述逐项对照人设图/分镜图
- 韩语台词保持原文，不翻译

### 参考图组织
每个场景准备 `场景X_图1_人设.png` + `场景X_图2_分镜.png`

## 固定规则

### 画风
韩式厚涂、暗朦调、泛朦、低饱和、反射、质感、泛光模糊晕染

### 图片生成
- 一律使用 image edit API（`/v1/images/edits`），以参考图为输入
- 禁止纯文本生成（角色不一致）
- 人设图和分镜图均需4K
- api2img代码模板见 `MM-KR-参考/api2img-setup.md`

### 角色表现
- 服装描述必须逐项对照人设图，禁止凭记忆
- 禁止夸张瞪眼等AI味浓的表情
- 表情自然克制

### 文档
- 每个视频一个完整文档，不分文档
- 中文文档，韩语台词+中文翻译
- 交付文件夹命名：`YYYY年M月D日_题材·关键词`

### Seedance提示词
- 提示词和参考图放同一文件夹
- [图1][图2]引用格式，禁止@图片1@图片2（版本C中@image1/@image2与之对应）
- 禁止引用拟声词音节（Seedance会当对白朗读）
- 镜头自然晃动模拟手持拍摄
- 时长按韩语台词音节计算（6-8音节/秒），目标8-12秒，单条上限15秒

### UI
- UI绿幕不带文案，可复用
- 提取固定提示词不可修改

### DeepWhite集成
- 编剧版（Phase 1版本B）：山音+screenwriter引擎，量化评分，因果审计
- 图片提示词（Phase 2&3附加）：双语提示词文档，供跨平台使用
- 分镜提示词（Phase 5版本C）：风格四段+镜头段落+微表演+摄影机-情绪同步，≤2200字
- 参考文件：`MM-KR-参考/deepwhite-编剧引擎.md`、`deepwhite-图片提示词模板.md`、`deepwhite-分镜提示词模板.md`
