# DeepWhite 编剧引擎（MM-KR适配版）

来源：deepwhite-screenwriting-v1，整合山音超级编剧大师 + screenwriter审计引擎。

## 适用位置

Phase 1 · 版本B，与原版简案并列输出，用户选择。

## 最高规则

- 只用中文输出
- 只写能被摄影机拍到、麦克风听到的内容
- 不写心理描写、括号暗示、解释性台词、说教独白、AI腔
- 台词口语化，像真人说话
- 用动作代替解释
- 对话像冰山，只露一角

## MM-KR体量路由

韩式买量素材通常为1-3分钟概念短片，走山音创作引擎的概念超短片路线：

- 概念在前5秒建立规则或制造好奇
- 角色为功能性archetype，不追求完整弧光
- 聚焦1-2种核心视听手法
- 台词极少，不用台词解释概念
- 前15秒必须有钩子

## 创作流程（精简版）

### 1. 破题与核心动作
- 戏剧动作 = 目标 + 阻碍
- 确定高概念和体量

### 2. 人物深度
- Want / Need / Arc
- 好感度走向（数值三段递进）
- 人物矛盾性

### 3. 结构大纲
- 开场钩子（前5-15秒抓人）
- 激励事件
- 递进冲突（每次比前一次更难）
- 高潮（主角做出的最终行动）
- 结尾钩子（让观众想看下一条）

### 4. 场景拆解
每场标注：
- 场景目标 / 阻碍 / 结果
- 价值转变（进入→退出）
- 双轨节奏（情节松紧 × 情感轻重）
- 预估时长

### 5. 场景写作
- 只写画面、动作、声音和口语化潜台词
- 情绪用视觉化写作（物理动作/物件/声音/停顿/距离/视线）

## 量化评分（1-3分钟概念片/AI视频）

```
开场钩子：/10
视觉概念：/10
场景因果：/10
人物主动性：/10
台词/VO克制：/10
结尾钩子：/10
3分钟节奏：/10
```

## screenwriter审计要点

### 控制性理念
```
生活变成 X，因为主角做了 Y，而原因是 Z。
```

### 场景价值转变
每场必须回答：开始时哪个价值处于什么状态？结束时变成什么？是什么动作造成转变？

常见价值对：控制/失控、信任/背叛、爱/占有、希望/绝望

### 因果性审计
逐场检查：第N场发生，是因为第N-1场发生了什么？好结构是"因为"，不是"然后"。

## 山音自检清单

- [ ] 是否建立目标+冲突？
- [ ] 开场是否有钩子？
- [ ] 激励事件是否明确？
- [ ] 高潮是否明确？
- [ ] 每段是否推动情节或揭示性格？
- [ ] 是否有心理描写？
- [ ] 是否有解释性台词？
- [ ] 台词是否口语化？
- [ ] 双轨节奏是否有起伏？

## 输出格式

```
【整体判断】...
【量化评分】...
【主要问题】...
【修改优先级】...
```
